1Appendix A. Vector Operations

2This appendix implements a class called 2Vector14141 that encapsulates all of the vector operations that you need when writing 2D or 3D rigid-body simulations. 3 Although 3Vector3 represents 3D vectors, you can easily reduce it to handle 2D vectors by eliminating all of the 3z3 terms or simply constraining the 3z14142 terms to 0 where appropriate in your implementation.

5Vector Class

6The 6Vector6 class is 8defined with three components—8x8, 8y8, and 8z14147 —along with several methods and operators that implement basic vector operations. 9 The class has two constructors, one of which initializes the vector components to 0, and the other of which initializes the vector components to those passed to the constructor:

//------------------------------------------------------------------------
// Vector Class and vector functions
//------------------------------------------------------------------------
class Vector {
public:
     float x;
     float y;
     float z;

     Vector(void);
     Vector(float xi, float yi, float zi);

     float Magnitude(void);
     void  Normalize(void);
     void  Reverse(void);

     Vector& operator+=(Vector u);
     Vector& operator-=(Vector u);
     Vector& operator*=(float s);
     Vector& operator/=(float s);

     Vector operator-(void);

};

// Constructor
inline Vector::Vector(void)
{
     x = 0;
     y = 0;
     z = 0;
}

// Constructor
inline Vector::Vector(float xi, float yi, float zi)
{
     x = xi;
     y = yi;
     z = zi;
}

65Here's the code that converts our 65Vector65 class vector to a unit vector:

inline     void  Vector::Normalize(void)
{
     float m = (float) sqrt(x*x + y*y + z*z);
     if(m <= tol) m = 1;
     x /= m;
     y /= m;
     z /= m;

     if (fabs(x) < tol) x = 0.0f;
     if (fabs(y) < tol) y = 0.0f;
     if (fabs(z) < tol) z = 0.0f;
}

float    const    tol = 0.0001f;

inline     Vector& Vector::operator*=(float s)

     y *= s;

     return *this;

163Conjugate: The − Operator

204Vector Cross Product: The ^ Operator

206This cross-product operator 210takes the vector cross product between vectors 210u210 and 210v210 (210u210 × 210v210) and returns a vector perpendicular to both 210u210 and 210v210 according to the formula:

210u210 × 210v210 = (u210y210 * v210z210 − u210z210 * v210y210) 210i210 + (−u210x210 * v210z210 + u210z210 * v210x210) 210j210 + (u210x210 * v210y210 − u210y210 * v210x210) 210k

212The resulting vector is perpendicular to the plane that contains vectors 212u212 and 212v14351 . 213 The direction in which this resulting vector points can be determined by the 213right hand rule14352 . 214 If you place the two vectors 214u214 and 214v214 tail to tail, as shown in 215Figure A-7215, and curl your fingers (of your right hand) in the direction from 215u215 to 215v14355 , then your thumb will point in the direction of the resulting vector.

222Figure A-7. Vector cross product

224In this case, the resulting vector points out of the page along the z-axis since the vectors 224u224 and 224v14363 lie in the plane formed by the x- and y-axes.

14366If two vectors are parallel, then their cross product will be 0. 14367 This is useful when you need to determine whether or not two vectors are indeed parallel.

231The cross-product operation is distributive; however, it is not commutative:

231u231 × 231v ≠ v231 × 231u
231u231 × 231v = −231(231v231 × 231u231)
231s (231u231 × 231v231) = (s)(231u231) × 231v231 = 231u231 × (s)(231v231)
231u231 × ( 231v + p231) = (231u231 × 231v231) + (231u231 × 231p231)

233Here's the code that takes the cross product of vectors 233u233 and 233v233:

inline     Vector operator^(Vector u, Vector v)
{
     return Vector(     u.y*v.z - u.z*v.y,
                         -u.x*v.z + u.z*v.x,
                         u.x*v.y - u.y*v.x );
}

14374Vector cross products are handy when you need to find normal (perpendicular) vectors. 14375 For example, when performing collision detection you often need to find the vector normal to the face of a polygon. 14376 You can construct two vectors in the plane of the polygon using the polygon's vertices and then take the cross product of these two vectors to get normal vector.

14403Vector dot products are handy when you need to find the magnitude of a vector projected onto another one. 14404 Going back to collision detection as an example, you often have to determine the closest distance from a point, which may be a polygon vertex on one body (body 1) to a polygon face on another body (body 2). 14405 If you construct a vector from the face under consideration on body 2, using any of its vertices, to the point under consideration from body 1, then you can find the closest distance of that point from the plane of body 2's face by taking the dot product of that point with the normal vector to the plane. 267 (If the normal vector is not of unit length, then you'll have to divide the result by the magnitude of the normal vector.)
